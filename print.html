REST API

Authentication

The API uses the open standard and stateless authentication method JSON Web Token (JWT). A token contains all the data required for authentication, so no database query is needed.

To learn more visit the official website.

Get A Token

If you do not have a token yet, you can apply for one using the following form.

API Request Form

Use A Token

For every request, a valid token must be transmitted. This can be done with passing the token via the authorization header with the type Bearer.

Authorization: Bearer <TOKEN>

Refresh A Token

Every token has a fixed lifetime of 30 minutes, after which time it must be refreshed. To get a new fresh token, a GET request to the /token endpoint must be made with a previous valid token.

There is a possibility that the lifetime of tokens will change in the future. Therefore, it's recommended to automatically check the expiration time of each token

GET /token

201 Created

Token successfully created

{
    "token": "<TOKEN>"
}

401 Unauthorized

No or invalid token

{
    "status": "Unauthorized",
    "message": "<ERROR MESSAGE>",
    "code": 401
}

403 Forbidden

Locked user

{
    "status": "Forbidden",
    "message": "User is locked",
    "code": 403
}

422 Unprocessable Entity

Token signing error

{
    "status": "Unprocessable Entity",
    "message": "<ERROR MESSAGE>",
    "code": 422
}
